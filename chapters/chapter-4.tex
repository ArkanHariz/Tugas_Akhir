\subsection{\textit{Preprocessing} Data} \label{IV.Preprocessing_Dataset}
Pada tahap ini dataset dilakukan \textit{preprocessing} sebelum dataset 
masuk ke model untuk dilakukannya proses \textit{training}. \textit{Preprocessing} 
yang dilakukan yaitu \textit{resize} dan normalisasi nilai citra. \textit{Resize} 
dilakukan untuk mengubah resolusi citra dari 28x28 piksel menjadi 224x224 piksel. 
Proses ini dilakukan untuk menyesuaikan inputan model yang dimana \textit{ResNet} 
menerima inputan citra dengan resolusi 224x224 piksel. Selain itu, alasan 
citra dilakukan \textit{resize} menjadi 224x224 piksel dikarenakan ingin 
melakukan perbandingan hasil dengan hasil yang model yang di-\textit{training} 
dengan resolusi citra 28x28 piksel. \par 

Kemudian dilakukannya normalisasi nilai citra untuk mengubah rentang nilai 
piksel dari [0, 255] menjadi [0, 1]. Normalisasi dilakukan supaya saat proses 
\textit{training} stabil. Normalisasi menggunakan nilai mean dan standar 
deviasi dari ImageNet yaitu [0.485, 0.456, 0.406] dan [0.229, 0.224, 0.225]. 

\begin{lstlisting}[caption={Kode Normalisasi dan Resize}, label={code:4.kode_normalisasi_resize}]
data_mean = [0.485, 0.456, 0.406]
data_std = [0.229, 0.224, 0.225]

def get_base_transform(size):
    return transforms.Compose([
        transforms.Resize((size, size)),
        transforms.ToTensor(),
        transforms.Normalize(mean=data_mean, std=data_std)
    ])
\end{lstlisting}


\subsection{Augmentasi Data} \label{IV.Augmentasi_Dataset}
Dataset yang sudah melewati \textit{preprocessing}, dataset di-augmentasi 
untuk memperbanyak jumlah data dan model dapat belajar dari berbagai jenis 
variasi citra. Augmentasi dilakukan hanya data pada kelas minoritas saja. 
Augmentasi yang dilakukan pada penelitian ini yaitu \textit{flip} (horizontal 
dan vertikal), \textit{rotation}, \textit{color jitter}, \textit{random erasing}, 
dan \textit{random augment}. Penerapan augmentasi ini bertujuan untuk 
mengurangi ketidakseimbangan distribusi kelas sehingga model tidak 
cenderung bias terhadap kelas mayoritas. Dengan variasi transformasi 
tersebut, model diharapkan mampu mempelajari fitur terhadap perubahan 
orientasi, pencahayaan, maupun gangguan visual pada citra. \par

\begin{lstlisting}[caption={Kode Augmentasi Data}, label={code:4.kode_augmentasi}]
def get_aug_transform(size):
    return transforms.Compose([
        transforms.Resize((size, size)),
        transforms.RandAugment(num_ops=2, magnitude=9),
        transforms.RandomHorizontalFlip(p=0.5),
        transforms.RandomVerticalFlip(p=0.5),
        transforms.RandomRotation(degrees=20),
        transforms.ColorJitter(brightness=0.3, contrast=0.3, saturation=0.3, hue=0.1),

        transforms.ToTensor(),

        transforms.RandomErasing(p=0.5, scale=(0.02, 0.15)),

        transforms.Normalize(mean=data_mean, std=data_std)
    ])
\end{lstlisting}

\subsubsection{Hasil \textit{ResNet-18} Menggunakan Citra \textit{Size} 28x28 Piksel} \label{IV.Resnet18_28}
Pada model \textit{ResNet-18 scratch} ini di \textit{training} menggunakan citra 
berukuran 28x28 piksel. Model yang digunakan merupakan model \textit{scratch}, 
yang dimana model menginisialisasi bobot secara acak. Hal ini berarti 
proses pembelajaran dilakukan sepenuhnya berdasarkan data penelitian tanpa 
memanfaatkan bobot \textit{pretrained}. Struktur kode model dapat dilihat pada 
Kode \ref{code:4.kode_resnet18_28} dan hasil evaluasi disajikan pada Tabel \ref{tab:hasil_resnet18_28}.

\begin{lstlisting}[caption={Kode ResNet-18 28x28}, label={code:4.kode_resnet18_28}]
class BasicBlock(nn.Module):
    expansion = 1
    def __init__(self, in_planes, planes, stride=1):
        super(BasicBlock, self).__init__()
        self.conv1 = nn.Conv2d(in_planes, planes, kernel_size=3, stride=stride, padding=1, bias=False)
        self.bn1 = nn.BatchNorm2d(planes)
        self.conv2 = nn.Conv2d(planes, planes, kernel_size=3, stride=1, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(planes)
        self.shortcut = nn.Sequential()
        if stride != 1 or in_planes != self.expansion*planes:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_planes, self.expansion*planes, kernel_size=1, stride=stride, bias=False),
                nn.BatchNorm2d(self.expansion*planes)
            )
    def forward(self, x):
        out = F.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out += self.shortcut(x)
        out = F.relu(out)
        return out
\end{lstlisting}

\subsubsection{Hasil \textit{ResNet-50} Menggunakan Citra \textit{Size} 28x28 Piksel} \label{IV.Resnet50_28}
Pada model \textit{ResNet-50 scratch} ini di \textit{training} menggunakan 
citra 28x28 piksel. Struktur kode model dapat dilihat pada Kode \ref{code:4.kode_resnet50_28} dan hasil 
evaluasi disajikan pada Tabel \ref{tab:hasil_resnet50_28}. \par

\begin{lstlisting}[caption={Kode ResNet-50 28x28}, label={code:4.kode_resnet50_28}]
class Bottleneck(nn.Module):
    expansion = 4
    def __init__(self, in_planes, planes, stride=1):
        super(Bottleneck, self).__init__()
        self.conv1 = nn.Conv2d(in_planes, planes, kernel_size=1, bias=False)
        self.bn1 = nn.BatchNorm2d(planes)
        self.conv2 = nn.Conv2d(planes, planes, kernel_size=3, stride=stride, padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(planes)
        self.conv3 = nn.Conv2d(planes, self.expansion * planes, kernel_size=1, bias=False)
        self.bn3 = nn.BatchNorm2d(self.expansion*planes)
        self.shortcut = nn.Sequential()
        if stride != 1 or in_planes != self.expansion*planes:
            self.shortcut = nn.Sequential(
                nn.Conv2d(in_planes, self.expansion*planes, kernel_size=1, stride=stride, bias=False),
                nn.BatchNorm2d(self.expansion*planes)
            )
    def forward(self, x):
        out = F.relu(self.bn1(self.conv1(x)))
        out = F.relu(self.bn2(self.conv2(out)))
        out = self.bn3(self.conv3(out))
        out += self.shortcut(x)
        out = F.relu(out)
        return out
\end{lstlisting}

\begin{lstlisting}[caption={Perbedaan Model \textit{Scratch}}, label={code:4.perbedaan_scratch}]
# Model Baseline (Scratch)
def forward(self, x):
    out = F.relu(self.bn1(self.conv1(x)))
    out = self.layer1(out)
    out = self.layer2(out)
    out = self.layer3(out)
    out = self.layer4(out)
    out = self.avgpool(out)

#-------------------------------------------------------------#

# Model Sendiri (Scratch)
def forward(self, x):
    out = F.relu(self.bn1(self.conv1(x)))
    out = self.layer1(out)
    out = self.layer2(out)
    out = self.layer3(out)
    out = self.layer4(out)
    out = self.avgpool(out)
    out = out.view(out.size(0), -1)
    out = self.dropout(out) # Dropout
    out = self.linear(out)
    return out
\end{lstlisting}

Pada \textit{Baseline}, model tidak menggunakan \textit{dropout} sehingga 
seluruh neuron pada setiap layer berkontribusi penuh 
saat proses \textit{training}. Berbeda dengan model sendiri yang 
menggunakan \textit{dropout} yang diletakka sesudah \textit{pooling} dan 
sebelum \textit{fully connected}. \textit{Dropout rate} yang digunakan 
pada model \textit{scratch} yaitu 0.7, yang berarti 70\% neuron akan 
dinonaktifkan secara acak pada setiap iterasi \textit{training}. 
Perlu diingat bahwa model ini dilatih dari awal (\textit{scratch}) dan 
memiliki dataset yang tidak seimbang, \textit{Dropout rate} dibuat cukup 
ekstrim ini bertujuan untuk menekan risiko overfitting \cite{srivastava2014dropout}.

\subsection{Penggunaan \textit{Class Weight}} \label{IV.Penggunaan_Class_Weight}
Pada penelitian ini, digunakannya \textit{class weight} yang diletakkan 
pada \textit{loss function} yaitu \textit{cross-entropy loss}. Penggunaan 
dapat \textit{class weight} dapat dilihat pada Kode \ref{code:4.class_weight}. \par

\begin{lstlisting}[caption={Penggunaan \textit{Class Weight}}, label={code:4.class_weight}]
counts = np.array([327, 514, 1099, 115, 1113, 6705, 142])
weights = 1.0 / counts
weights = weights / weights.sum() * n_classes 
class_weights = torch.tensor(weights).float().to(device)

criterion = nn.CrossEntropyLoss(weight=class_weights)
\end{lstlisting}

Penggunaan \textit{class weight} pada penelitian ini bertujuan untuk 
menangani \textit{imbalanced data}. Berdasarkan pada Kode \ref{code:4.class_weight}, 
menghitung nilai invers dari jumlah data pada masing-masing kelas untuk 
menentukan bobot yang berguna untuk memberikan penalti lebih besar saat 
model salah memprediksi pada kelas minoritas. Proses ini akan memberikan 
bobot tertinggi pada kelas dengan jumlah data nya paling sedikit sementara 
kelas yang memiliki jumlah data nya paling banyak mendapatkan bobot yang 
rendah. \textit{Class weight} digunakan pada \textit{loss function} yaitu \textit{cross-entropy loss} 
supaya model dapat setara terhadap semua kelas.

dan mampu deteksi data yang benar dari keseluruhan data pada kelas nya masing-masing 
(\textit{recall}). Sementara untuk model \textit{ResNet-18} dengan citra 224x224 
yang tidak melakukan augmentasi minoritas memiliki hasil yang rendah yang 
membuat model tersebut masih salah dalam mendeteksi data yang benar dari 
keseluruhan data pada kelas nya masing-masing berdasarkan nilai \textit{recall}-nya 
yang berada pada angka 0.53. Selain hasil \textit{ResNet-18} dengan citra 
224x224 augmentasi minorits yang unggul, model dapat mengurangi risiko 
overfitting yang mana grafik \textit{val loss} yang menurun seiring 
berjalanya proses \textit{training} walaupun terdapat lonjakan dibeberapa
epoch dibandingkan \textit{ResNet-18} 224x224 
tanpa melakukan augmentasi minoritas yang masih \textit{overfitting} yang ditandai 
\textit{val loss} yang tidak menurun sampai akhir proses \textit{training}. 
Untuk grafik \textit{loss} \textit{Resnet-18} 224x224 augmentasi minoritas 
dapat dilihat pada Gambar \ref{fig:4.resnet18_224_aug} dan grafik los \textit{Resnet-18} 224x224 tanpa 
augmentasi minoritas (\textit{baseline}) dapat dilihat pada Gambar \ref{fig:4.resnet18_224_no_aug}. \par

Untuk model \textit{ResNet-18} dengan citra 28x28 yang dilakukan augmentasi 
minoritas, hasil evaluasi model masih belum melebihi model \textit{baseline} 
yang tidak melakukan augmentasi minoritas. Dari segi akurasi, model 
\textit{ResNet-18} dengan augmentasi minoritas hanya mendapatkan akurasi 
sebesar 0.60 yang mana lebih rendah dibandingkan model \textit{baseline} 
yang mendapatkan akurasi sebesar 0.75. Nilai \textit{precision} makro pun 
hanya mendapatkan 0.44 berbanding jauh dengan model \textit{baseline} 
yang mendapatkan \textit{precision} 0.62. Dikarenakan nilai \textit{precision} 
yang rendah pada model \textit{ResNet-18} augmentasi minoritas, membuat 
nilai \textit{f1-score} mendapatkan nilai yang rendah yaitu 0.45. Meskipun 
pada model \textit{baseline}, nilai \textit{f1-score} nya mendapatkan 0.54 
yang mana masih belum cukup baik tetapi dibandingkan dengan \textit{ResNet-18} 
dengan augmentasi minortias, nilai \textit{f1-score} pada model 
\textit{baseline} lebih baik. Disisi lain, dapat dilihat pada \textit{loss}
